# Design Principles & Patterns

## 📌 Purpose

Practice exercises implementing classic **Gang of Four design patterns** in plain Java — the **Singleton** pattern and the **Factory Method** pattern — applied to realistic scenarios (a shared logger and a document-opening application).

## 📂 Files

| File | Description |
|------|-------------|
| `SingletonPatternExample.java` | Implements the **Singleton** pattern with a `Logger` class that has a private constructor and a static `getInstance()` method, guaranteeing only one instance ever exists. |
| `FactoryMethodPatternExample.java` | Implements the **Factory Method** pattern with a `Document` interface, three concrete documents (`WordDocument`, `PdfDocument`, `ExcelDocument`), and corresponding factories (`WordFactory`, `PdfFactory`, `ExcelFactory`). |
| `*.class` | Compiled bytecode output from `javac`. These are build artifacts, not source files (see note below). |

## 🔍 File Details

### `SingletonPatternExample.java`
- `Logger` — has a `private static Logger instance`, a `private` constructor, and `public static Logger getInstance()` which lazily creates the single instance.
- `log(String message)` prints a formatted log line.
- `main` retrieves the `Logger` instance twice and confirms (via `==` and `hashCode()`) that both references point to the **same object**.

### `FactoryMethodPatternExample.java`
- `Document` — interface with an `open()` method.
- `WordDocument`, `PdfDocument`, `ExcelDocument` — concrete implementations that print which document type is opening.
- `DocumentFactory` — abstract factory with `createDocument()`.
- `WordFactory`, `PdfFactory`, `ExcelFactory` — concrete factories, each returning their respective document type.
- `main` demonstrates creating and opening each document type via its factory.

## 📦 Dependencies

None — both files use only the Java standard library.

## ▶️ Usage

```bash
cd "JAVA FS Engineer/Engineering concepts/Design principles & Patterns"

# Singleton pattern
javac SingletonPatternExample.java
java SingletonPatternExample
# Output:
# Logger Instance Created
# LOG: Application Started
# LOG: Processing Data
# logger1 HashCode: <hash>
# logger2 HashCode: <hash>  (same as logger1)
# Only one Logger instance exists.

# Factory Method pattern
javac FactoryMethodPatternExample.java
java FactoryMethodPatternExample
# Output:
# Opening Word Document
# Opening PDF Document
# Opening Excel Document
```

> **Note:** The compiled `.class` files in this folder are build artifacts generated by `javac`. They can be safely deleted and regenerated, and are good candidates for a `.gitignore` entry (e.g. `*.class`) to keep the repository source-only.
